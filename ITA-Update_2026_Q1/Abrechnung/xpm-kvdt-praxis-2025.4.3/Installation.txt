Dieses Paket enthält das neue KBV-Prüfmodul für die KVDT-Prüfung in der Arztpraxis.


Im obersten Verzeichnis dieses Paketes existiert der Ordner 'XPM_KVDT.Praxis'.

Vor dem Starten des Prüfmoduls muss der komplette Ordner 'XPM_KVDT.Praxis'
auf eine Festplatte kopiert werden.

Das KBV-Prüfmodul kann nur gestartet werden, wenn eine Java Laufzeitumgebung
in der Version 17 oder höher bereits installiert wurde.

Unter Linux ist noch folgendes zu beachten: Bei Verwendung einer auf OpenJDK 
basierenden Laufzeitumgebung und einer Servervariante eines Linux-Betriebssystems 
muss zusätzlich die Bibliothek fontconfig installiert werden bzw. bereits 
installiert sein. Das ist notwendig, da die fontconfig.properties Datei für Linux 
im OpenJDK im Gegensatz zur Oracle-Variante nicht enthalten ist.
Wenn eine Desktop-Variante von Linux oder eine Oracle-Laufzeitumgebung verwendet 
wird, muss die Bibliothek nicht installiert werden.


Die Konfiguration der Prüfumgebung braucht nicht angepasst werden.

In diesem Verzeichnis befinden sich Batchdateien bzw. Shellskripte,
die das KBV-Prüfmodul mit unterschiedlichen Optionen ausführen.

Die Batchdatei 'StartPruefung.bat' bzw. das Shellskript 'StartPruefung.sh'
startet eine Prüfung einer einzigen Datei. Andere Abrechnungsdateien können
geprüft werden, indem die Batchdatei entsprechend angepasst wird (Übergabeparameter -f).

Die Batchdatei 'StartPruefmodul.bat' bzw. das Shellskript 'StartPruefmodul.sh'
startet eine grafische Oberfläche. Die Prüfung kann hier über den Knopf 'Starten'
gestartet werden.

Die Batchdatei 'TesteAusgaben.bat' bzw. das Shellskript 'TesteAusgaben.sh' demonstriert
die diversen Ausgabemöglichkeiten des neuen KBV-Prüfmoduls.
  
Bitte lesen Sie auch das Dokument 'KBV_ITA_AHEX_Handbuch_Pruefmodul.pdf'.
