Dieses Paket enthält das neue LDK-Prüfmodul für die Prüfung der LDT 3.2.19 Schnittstelle sowie der digitalen Muster 10 und 10A.

Im obersten Verzeichnis dieses Paketes existiert der Ordner 'XPM_LDK'.

Vor dem Starten des Prüfmoduls muss der komplette Ordner 'XPM_LDK'
auf eine Festplatte kopiert werden.

Das LDK-Prüfmodul kann nur gestartet werden, wenn eine Java Laufzeitumgebung
in der Version 17 oder höher bereits installiert wurde.

Unter Linux ist noch folgendes zu beachten: Bei Verwendung einer auf OpenJDK 
basierenden Laufzeitumgebung und einer Servervariante eines Linux-Betriebssystems 
muss zusätzlich die Bibliothek fontconfig installiert werden bzw. bereits 
installiert sein. Das ist notwendig, da die fontconfig.properties Datei für Linux 
im OpenJDK im Gegensatz zur Oracle-Variante nicht enthalten ist.
Wenn eine Desktop-Variante von Linux oder eine Oracle-Laufzeitumgebung verwendet 
wird, muss die Bibliothek nicht installiert werden.


Die Konfiguration der Prüfumgebung braucht nicht angepasst werden.

In diesem Verzeichnis befinden sich Batchdateien bzw. Shellskripte, die das LDK-Prüfmodul
mit unterschiedlichen Optionen ausführen.

Die Batchdatei 'StartPruefung.bat' bzw. das Shellskript 'StartPruefung.sh'
starten eine Prüfung einer einzigen Datei. Andere Dateien können geprüft werden,
indem die Batchdatei entsprechend angepasst wird (Übergabeparameter -f).

Die Batchdatei 'StartPruefmodul.bat' bzw. das Shellskript 'StartPruefmodul.sh'
starten eine grafische Benutzeroberfläche.

Die Batchdatei 'VerzeichnisPruefung.bat' bzw. das Shellskript 'VerzeichnisPruefung.sh'
startet das LDK-Prüfmodul im Servermodus (Übergabeparameter -s).
Das LDK-Prüfmodul verarbeitet alle Dateien, die im Ordner 'Daten' sind.
Je nach Prüfstatus werden die Dateien anschliessend in die Ordner
'Abgelehnt' bzw. 'NichtAbgelehnt' verschoben (Übergabeparameter -m).
Die Prüfung wird beendet, wenn alle Dateien verarbeitet wurden (Übergabeparameter -e)
